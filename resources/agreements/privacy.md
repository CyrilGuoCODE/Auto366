# Auto366 用户隐私协议

---

## 1. 引言

欢迎使用 Auto366（以下简称"本软件"）。本软件是一款开源的 HTTP 代理工具，主要功能包括 HTTP/HTTPS 代理、规则匹配与响应修改、答案提取与上传等。

我们重视您的隐私。本协议说明了我们如何收集、使用、存储和保护您的信息。使用本软件即表示您同意本协议中描述的数据处理实践。

**本软件是开源项目，您可以在 [GitHub 仓库](https://github.com/cyrilguocode/Auto366) 查看完整源代码，验证我们的数据处理行为。**

---

## 2. 数据收集与分析

### 2.1 PostHog 用户数据分析

本软件集成了 [PostHog](https://posthog.com/) 用于产品使用分析，帮助我们了解用户如何使用本软件，从而改进产品体验。

#### 我们收集的数据

所有事件均附带以下自动附加属性：

| 属性 | 说明 |
|------|------|
| `app_version` | 当前软件版本号（如 `0.8.5`） |
| `platform` | 操作系统类型（`win32`、`darwin`、`linux`） |
| `distinctId` | 随机生成的匿名设备标识符（UUID v4，与任何个人身份无关） |

#### 具体收集事件列表

**应用生命周期事件：**

| 事件名 | 触发时机 | 附加属性 |
|--------|----------|----------|
| `app_launched` | 应用启动时 | 无 |
| `app_initialized` | 前端界面初始化完成时 | 无 |
| `app_closed` | 应用关闭时 | 无 |

**代理功能事件：**

| 事件名 | 触发时机 | 附加属性 |
|--------|----------|----------|
| `proxy_start_requested` | 用户点击"启动代理"按钮时 | 无 |
| `proxy_started` | 代理服务器成功启动时 | 无 |
| `proxy_stopped` | 代理服务器停止时 | 无 |

**答案提取事件：**

| 事件名 | 触发时机 | 附加属性 |
|--------|----------|----------|
| `answer_extracted` | 从 ZIP 文件中成功提取到答案时 | `count`：提取的答案数量 |

**规则触发事件：**

| 事件名 | 触发时机 | 附加属性 |
|--------|----------|----------|
| `zip_implant_applied` | ZIP 注入规则被触发时 | `rule_type`：`fileinfo`（文件信息修改）或 `download`（文件下载替换） |
| `dynamic_inject_applied` | 动态注入规则被触发时 | `phase`：`fileinfo`（文件信息阶段）或 `download`（下载阶段） |
| `answer_upload_applied` | 答案上传规则被触发时 | `upload_type`：`original`（原始数据）或 `extracted`（提取的答案） |

**界面操作事件：**

| 事件名 | 触发时机 | 附加属性 |
|--------|----------|----------|
| `view_switched` | 用户切换视图页面时 | `view`：目标页面（`answers`/`rules`/`community`/`settings`）；`ui_mode`：当前模式（`simple`/`professional`） |
| `ui_mode_switched` | 用户切换 UI 模式时 | `mode`：`simple`（简易模式）或 `professional`（专业模式） |

**设置变更事件：**

| 事件名 | 触发时机 | 附加属性 |
|--------|----------|----------|
| `analytics_enabled` | 用户开启数据分析时 | 无 |

#### 我们**不**收集的数据

- 姓名、邮箱、手机号等个人身份信息
- 密码、API Key 等敏感凭证
- 浏览的网页内容或具体网址
- 代理处理的请求/响应内容
- 规则配置的具体内容（规则名称、URL 匹配模式等）
- 答案文件的具体内容
- ZIP 注入文件的具体内容
- 缓存路径、端口等用户配置信息
- 任何用户主动输入的数据

### 2.2 数据使用目的

我们收集数据仅用于以下目的：

- 识别常见错误和崩溃，提高软件稳定性
- 统计功能使用频率，优先开发用户需求
- 了解代理和规则功能的实际使用情况，改进核心功能
- 了解用户界面操作习惯，优化界面设计
- 统计活跃用户数和版本分布，评估项目健康状况

## 3. 数据处理与存储

### 3.1 数据控制者

本软件项目的维护者作为数据控制者，负责决定收集哪些数据以及如何使用这些数据。

### 3.2 数据处理者

[PostHog](https://posthog.com/) 作为数据处理者，帮助我们处理和分析收集到的使用数据。PostHog 是独立的第三方服务提供商，拥有自己的隐私政策和安全措施。

### 3.3 数据存储位置

- 使用数据通过加密连接（HTTPS）传输至 PostHog 服务器
- 数据在 PostHog 平台上的存储期限由我们设定，最长不超过 12 个月

### 3.4 本地存储

以下数据存储在用户本地设备上，不会上传至任何服务器：

| 文件 | 路径 | 说明 |
|------|------|------|
| 匿名设备 ID | `~/.Auto366/.device_id` | 随机生成的 UUID，用于区分不同设备 |
| 分析开关状态 | `~/.Auto366/analytics.json` | 记录用户是否允许数据分析 |

## 4. 用户权利与选择

### 4.1 退出数据分析

您随时可以选择退出用户数据分析：

1. 打开软件设置界面
2. 找到"数据分析"选项
3. 关闭"允许发送使用统计数据"复选框

退出后，本软件将不再向 PostHog 发送任何使用数据。已收集的数据将按照本协议规定的时间自动删除。

### 4.2 数据访问与删除

根据适用的数据保护法规（如 GDPR），您拥有以下权利：

- **访问权**：了解我们收集了哪些关于您的数据
- **删除权**：要求删除我们存储的关于您的数据
- **选择退出权**：随时停止数据收集

如需行使上述权利，请通过以下方式联系我们：

- GitHub Issues：[https://github.com/cyrilguocode/Auto366/issues](https://github.com/cyrilguocode/Auto366/issues)
- 或通过 GitHub 直接联系项目维护者

## 5. 数据安全

我们采取合理的技术和组织措施保护您的数据：

- 数据传输使用 HTTPS 加密
- 不收集个人身份信息，从源头降低隐私风险
- 每个事件仅包含最小必要信息（如答案数量而非答案内容）
- 定期审查数据收集代码，确保符合本协议声明
- PostHog 平台本身通过了 SOC 2 Type II 安全认证

---

## 6. 其他数据收集

### 6.1 自动更新检查

本软件会定期连接 GitHub 服务器检查新版本更新。此过程仅发送当前版本号，用于判断是否有可用更新，不会发送其他个人信息。

### 6.2 规则集加载

本软件可能从远程服务器加载社区规则集。此过程会发送请求以获取规则文件，不包含用户个人数据。
